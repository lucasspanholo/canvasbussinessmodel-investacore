Business Case

Necessidade de mercado: Investidores de criptomoedas precisam de uma ferramenta centralizada para gerenciar seus ativos digitais, acompanhar performance e tomar decisões informadas.


Oportunidade: Criar uma plataforma acessível que democratize o controle de investimentos em criptomoedas, oferecendo análises visuais claras e alertas automáticos para investidores de todos os níveis de experiência.


Valor econômico: Reduzir o tempo gasto em análises manuais, minimizar perdas por falta de monitoramento e maximizar oportunidades de investimento através de dados em tempo real.

Functional Overview
  • Cadastro de investidor: Registro com dados pessoais
  • Uso anônimo: Importação/exportação JSON
  • Registro de criptomoedas: Informações de compra (moeda, quantidade, data, valor)
  • Controle de vendas: Registro de saques/vendas parciais ou totais
  • Cotações em tempo real: Integração com APIs de mercado
  • Gráficos individuais: Variação por período (diário, semanal, mensal)
  • Gráfico consolidado: Evolução do portfólio total
  • Notificações automáticas: Alertas para variações ≥ 5%
Business Context

Usuários:

  • Investidores iniciantes e experientes
  • Usuários anônimos (teste via JSON)

Sistemas externos:

  • APIs de cotação (Binance, CoinGecko)
  • Serviços de notificação (email, push, SMS)
  • Navegadores web (interface de acesso)

Fluxo principal:

  • 1. Cadastro de ativos ou importação JSON
  • 2. Consulta automática de cotações
  • 3. Geração de gráficos e análises
  • 4. Envio de alertas automáticos
Organisational Constraints
  • OC1 - Equipe pequena: Foco em simplicidade e modularidade da arquitetura
  • OC2 - Tempo limitado: Funcionalidades principais devem ter prioridade
  • OC3 - Versionamento: Repositório Git com CI/CD obrigatório
  • OC4 - Licenciamento: Projeto deve ser Open Source
  • OC5 - Documentação: Padrão arc42 para documentação arquitetural
  • OC6 - Convenções: Código em inglês, documentação em português
Technical Constraints
  • TC1 - Backend: Implementação obrigatória em .NET para robustez e modularidade
  • TC2 - Frontend: SPA em React para experiência fluida e multiplataforma
  • TC3 - Persistência: Banco relacional (SQL Server) para consistência dos dados
  • TC4 - Integrações: APIs externas de cotação em tempo real (Binance, CoinGecko)
  • TC5 - Interoperabilidade: Suporte a exportação/importação JSON
  • TC6 - Deploy: Docker para facilitar desenvolvimento e distribuição
Quality Goals

1. Confiabilidade

O sistema deve calcular corretamente ganhos e perdas, sem inconsistências nos dados financeiros.


2. Usabilidade

Interface clara e intuitiva para que investidores de qualquer nível possam utilizá-la efetivamente.


3. Escalabilidade

Suportar aumento de usuários e ativos sem degradação de performance ou disponibilidade.

Architecture Hypotheses

H1 - Arquitetura Modular: Separação clara entre apresentação (React), lógica de negócio (.NET API) e persistência (SQL Server) permitirá manutenibilidade e testabilidade.


H2 - Integração via APIs REST: Comunicação HTTP/JSON entre frontend e backend garantirá interoperabilidade e facilidade de evolução.


H3 - Caching de Cotações: Cache de dados de APIs externas reduzirá latência e custos de requisições.


H4 - Uso Anônimo: Funcionalidade JSON permitirá testes sem cadastro, aumentando adoção inicial.

Technical Challenges & Risks

ALTO - Dependência de APIs Externas:

  • Rate limiting das APIs de cotação
  • Indisponibilidade temporária dos serviços
  • Mudanças nos contratos das APIs

MÉDIO - Precisão de Cálculos:

  • Arredondamentos em operações financeiras
  • Sincronização de dados em tempo real

MÉDIO - Performance e Escalabilidade:

  • Volume alto de requisições simultâneas
  • Processamento de grandes volumes de dados históricos
